## 2. 컨벤션
- 각 기능 또는 작업에 대한 새로운 브랜치를 생성하고, 완료 시 'main'에 병합
- 새로운 기능이나 버그 수정은 브랜치를 생성하고, 해당 브랜치에서 작업 후 Pull Request를 통해 'master'로 병합
- 브랜치 네이밍으로 이니셜/기능/구현하는 메소드명 형식으로 전환하여 사용 (ex. lsh/user/join)
